## Введение

Это руководство подойдёт:

* *Начинающим разработчикам*
* *Техническим писателям*
* *Специалистам смежных IT-профессий*

**Git** - это специальная программа, которая позволяет отслеживать любые изменения в файлах, хранить их версии и оперативно возвращаться в любое сохранённое состояние.

**Git Bash** - это эмулятор командной строки, который предоставляет пользователям Windows доступ к среде выполнения команд Git  с интерфейсом в стиле Unix.

**GitHub** - это крупнейший веб-сервис для хостинга IT-проектов и их совместной разработки. Веб-сервис основан на системе контроля версий Git.

## Установка Git и Git Bash для Windows

Перед началом работы необходимо установить Git, который включает в себя Git Bash - эмулятор командной строки для работы с Git.

### Инструкция по установке

1. **Скачайте установочный файл с официального сайта.**
   
    Перейдите на [официальный сайт Git](https://git-scm.com/). Нажмите на кнопку **Download for Windows**. Будет скачан установочный файл.

2. **Запустите установочный файл.**
   
   Дважды щелкните по скачанному файлу. Появится окно контроля учетных записей. Нажмите «Да», чтобы разрешить приложению вносить изменения.

3. **Настройте параметры установки.**

    Следуйте шагам мастера установки. Рекомендуется оставить настройки по умолчанию, но на некоторых шагах стоит обратить внимание:
    * **Select Components:**
     
      Оставьте все галочки по умолчанию. Убедитесь, что `Git Bash` и `Git GUI` выбраны.
  
      ![Select Components](/docs/img/SelectComponents.png)

    * **Choosing the default editor:**
      Если вы планируете писать код в **VSCode**, выберите **Visual Studio Code** из выпадающего списка. Если вы не уверены, оставьте **Vim**.
  
      ![Default Editor](/docs/img/DefaultEditor.png)

      > *Вы всегда можете изменить редактор позже с помощью команды*:
      `git config --global core.editor "code --wait"`.


    * **Adjusting the name of the initial branch in new repositories:** 
     
      Раньше Git по умолчанию создавал начальную ветку с именем `master`. Однако сейчас сообщество переходит на использование нейтрального имени `main`.
     
      Выберите `Override the default branch name` - задать свое имя `main`.

      ![Adjusting the name of the initial branch in new repositories](/docs/img/NameBranch.png)

    * **Adjusting your PATH environment:** 
     
      Выберите: `Git from the command line and also from 3rd-party software` - это самый гибкий вариант, позволяющий использовать Git в любом терминале.
  
      ![Adjusting your PATH environment](/docs/img/AdjustingEnvironme.png)

    * **Choosing the SSH executable:** 
     
      Git предложит выбрать, какой SSH-клиент использовать для работы с репозиториями по протоколу SSH. 
     
      SSH (Secure Shell) — это протокол для безопасного подключения к удаленным серверам.
     
       Выберите: `Use external OpenSSH` - стандартный OpenSSH, который идет в комплекте с Git для Windows
  
      ![Choosing the SSH executable](/docs/img/ssh.png)

    * **Choosing HTTPS transport backend:** 
      
      Git предложит выбрать бэкенд для HTTPS-соединений. Этот выбор влияет на то, как Git будет работать с защищенными репозиториями на GitHub и других хостингах.

      Оставьте галочки по умолчанию: `Use the OpenSSL library`.
    * **Configuring the line ending conversions:**
     
      Это важный пункт для совместимости между Windows и Linux/macOS. 
      
      Выберите: `Checkout Windows-style, commit Unix-style line endings`.

      ![Configuring the line ending conversions](/docs/img/Configuring.png)

    * **Configuring the terminal emulator to use with git bash:**</br>Git предложит выбрать эмулятор терминала для работы с Git Bash. Этот выбор влияет на то, как будет выглядеть и работать ваша командная строка.
     
      Выберите: `Use MinTTY (the default terminal of MSYS2)` - более привычный интерфейс для пользователей UNIX-подобных систем (Linux, macOS).
  
      ![Configuring the terminal emulator to use with git bash](/docs/img/ConfiguringEmulator.png)

    * **Choose the default behavior of git pull:**
    
      Git предложит выбрать стратегию по умолчанию для команды `git pull`. Этот выбор влияет на то, как Git будет объединять изменения из удаленного репозитория с вашей локальной веткой.
     
      Оставьте галочки по умолчанию: `Fast-forward or merge` - это безопасный вариант, который гарантирует сохранность ваших данных и создает понятную историю коммитов.
    * **Choose a credential helper:** 
     
      Git предложит выбрать способ хранения ваших учетных данных для автоматической аутентификации на GitHub и других хостингах.

      Оставьте галочки по умолчанию: `Git Credential Manager` - надежно сохраняет ваши данные для входа и избавляет от рутинного ввода пароля при каждой операции с GitHub. 

    * **Configuring extra options:**
  
      Git предложит несколько дополнительных опций, которые расширяют функциональность и улучшают производительность.
      
      Оставьте галочки по умолчанию: `Enable file system caching` - эта опция значительно ускоряет работу Git с большими репозиториями.

4. **Завершите установку**

   Нажмите `Install` и дождитесь окончания процесса. После этого нажмите `Finish`.

## Настройки Git 

Перед началом работы необходимо указать свое имя и почту. Эта информация будет использоваться для подписи ваших коммитов. При этом не имеет значения, в какой директории вы находитесь, вызов `git config --global` сработает везде.

```bash
git config --global user.name "Ваше Имя"
git config --global user.email "username@mail.ru"
```
*Примечание: Замените "Ваше Имя" и "username@mail.ru" на данные, которые вы используете на GitHub. Имя необходимо написать латиницей.*

Для проверки настроек выполните: `git config --list`.

Если вы сделали все верно командная строка покажет текущее значение.
```bash
user.name "Ваше Имя"
user.email "username@mail.ru"
```

## Инициализация репозитория

Преобразование обычной директории в репозиторий Git для начала контроля версий.

### Создание нового репозитория

Создайте папку с вашим проектом `my-project`. Перейдите в неё с помощью команд `cd` и выполните `git init`.

```bash
# перешли в нужную папку
cd ~/dev/my-project

# создали репозиторий 
git init 
```
Результат выполнения:
* Создается скрытая папка `.git` с внутренней структурой Git.
* Директория становится отслеживаемым репозиторием.
* Появляется возможность фиксировать изменения файлов.

### Проверка успешной инициализации репозитория

После инициализации необходимо проверить текущее состояние репозитория. Введите команду `git status`.

```bash
# Проверка статуса
git status
```
Ожидаемый вывод:

```bash
# название текущей ветки
On branch main

# сообщение о том, что в репозитории ещё нет коммитов
No commits yet

# Нет изменений для фиксации (создайте или скопируйте файлы и используйте "git add" для начала отслеживания)
nothing to commit (create/copy files and use "git add" to track)
```

## Добавление файлов в репозиторий

Перемещение файлов из рабочей директории в область подготовленных файлов для последующей фиксации в коммите.

### Основные команды добавления файлов

* Добавление отдельных файлов:
```bash
# Добавление одного конкретного файла
git add README.md

# Добавление нескольких конкретных файлов
git add index.html style.css

# Добавление файла из поддиректории
git add src/main.js
```
* Добавление групп файлов:
```bash
# Добавление всех файлов в текущей директории (включая поддиректории)
git add .

# Добавление всех изменений в проекте (включая удаленные файлы)
git add -A

# Добавление всех файлов с определенным расширением
git add *.js
git add *.css

# Добавление всех файлов в конкретной директории
git add images/
git add src/components/
```
После необходимо проверить состояние добавленных файлов командой `git status`.

```bash
# Ответ в командной строке
On branch main
No commits yet
Changes to be committed:
  (use "git rm --cached <file>..." to unstage)
    new file: # Имя файла(Файл подготовлен к коммиту)
    new file: # Имя файла(Файл подготовлен к коммиту)
    new file: # Имя файла(Файл подготовлен к коммиту)
```
## Просмотр изменений в файлах

Позволяет увидеть, что именно изменилось в файлах. Проверить код перед сохранением и найти ошибки перед коммитом.

  * Посмотреть неподготовленные изменения:
```bash
# Изменения в файлах, которые еще не добавлены через git add
git diff
```
  * Посмотреть подготовленные изменения:
```bash
# Изменения в файлах, которые уже добавлены через git add
git diff --staged
```
  * Сравнение конкретного файла:
```bash
# Только для README.md
git diff README.md

# Только подготовленные изменения для index.html
git diff --staged index.html
```
## Создание коммита

Коммит - это фиксация и сохранение в истории Git. При необходимости можно будет «откатиться» на другие версии проекта.

Команда коммита:
```bash
git commit -m "first commit" 
```
 ключ `-m` (от англ. message — «сообщение»), который позволяет добавить к коммиту сообщение.

 ### Просмотр информации о коммите

Команды для просмотра:
```bash
# Просмотр последнего коммита
git show

# Просмотр конкретного коммита
git show abc123def

# Компактный просмотр истории
git log --oneline
```
